# TOOLS.md - Какой инструмент когда

Правило выбора: к задаче подбирай инструмент по триггеру, не наугад.

| Триггер / что нужно | Инструмент | НЕ делай |
|---|---|---|
| данные, счётчики, состояние пайплайна, «сколько/что в БД» | SQL к `mvp_db` | не рассуждай, не угадывай числа |
| «есть ли код / файл / воркер», структура агента | `ls`/`grep` по `/root/.openclaw/workspace/agents/` | не утверждай наличие/отсутствие без проверки |
| LLM-разметка / обогащение / вызов агента | `openclaw agent --agent <name> ...` как в `label_kr_worker.py` | не звать LLM иначе |
| схема БД, имена колонок | миграции `db/*.sql` + `pg_catalog` | не выдумывать поля |
| зафиксировать факт/решение | дописать `_tracking/STATE.md` или `LOG.md`, затем `sync.sh` | не отчитываться без записи |

## Жёсткие запреты

- Не обращаться к `mcp__pg_research`, `mcp__pg_randd_res_task_tracker` и dzen-таблицам.
- Не модифицировать боевую очередь и таблицы `agent_1`, если задача этого явно не требует.
- Не запускать `parsers360_ingest` без явного указания: это поход в сеть.

## Окружение

- БД A1: DSN в `/root/.openclaw/workspace/agents/agent_1/.env` (`AGENT_1_DB_DSN`), БД `mvp_db`.
- Вызов LLM: внешний CLI `openclaw` в `PATH` (`/root/.hermes/node/bin/openclaw`).
