## ADDED Requirements

> Статус: эта спека форвард-лукинг, а не ретроактивная фиксация. По
> состоянию на 2026-07-07 ни один Python-воркер не реализует расширение
> аббревиатур или триграммную статистику отзывов — в коде
> (`agents/agent_1/src/agent_1/`, `agents/agent_1/scripts/`) такого воркера
> нет, а `corporate_abbreviations`/`review_trigrams` существуют только как
> пустые таблицы-заготовки в `db/001_init.sql`. Требования ниже описывают
> целевое поведение по `agents/agent_1/PIPELINE_V1.md`, которое ещё
> предстоит реализовать (см. `tasks.md`).

### Requirement: Разделение обязанностей — детерминированный слой, не агент
Расширение аббревиатур и триграммная статистика отзывов MUST выполняться
отдельным Python-воркером/скриптом, а не `agent_1` (LLM). `agent_1`
семантическому уровню (`agent_1-semantic-enrichment`) MUST NOT делегироваться
ни один из этих двух выходов.

#### Scenario: Попытка делегировать расширение аббревиатур агенту
- **WHEN** для документа требуется расширить обнаруженную аббревиатуру
- **THEN** это выполняет детерминированный Python-воркер по корпоративному
  словарю, а не LLM-вызов `agent_1`

### Requirement: Обнаружение и расширение аббревиатур
Воркер MUST детектировать кандидатов-аббревиатур в очищенном документе
детерминированными правилами/регулярными выражениями и расширять их только
по данным поддерживаемого корпоративного словаря (таблица
`corporate_abbreviations` с колонками `short`, `expanded`). Если для
обнаруженной аббревиатуры есть запись в словаре, результат MUST иметь
`status='resolved'` и заполненный `expanded`; если записи ещё нет —
`status='unresolved'` с `expanded=null`.

#### Scenario: Аббревиатура есть в словаре
- **WHEN** обнаруженная аббревиатура (например, `ABC`) присутствует в
  `corporate_abbreviations`
- **THEN** результат содержит `status='resolved'` и соответствующее
  значение `expanded`

#### Scenario: Аббревиатуры ещё нет в словаре
- **WHEN** обнаруженная аббревиатура отсутствует в `corporate_abbreviations`
- **THEN** результат содержит `status='unresolved'` и `expanded=null`, без
  ошибки и без блокировки обработки документа

### Requirement: Rerun-совместимость после пополнения словаря
Нерасширенные (`unresolved`) записи MUST оставаться повторно
обрабатываемыми: после добавления новой записи в корпоративный словарь
должен быть возможен backfill-прогон, который переводит ранее
`unresolved` записи в `resolved`, не требуя повторного прохождения
документа через агентские стадии (`agent_1-kr-labeling`,
`agent_1-semantic-enrichment`).

#### Scenario: Словарь пополнен после первого прогона
- **WHEN** в `corporate_abbreviations` добавлена запись для ранее
  `unresolved` аббревиатуры, и запускается backfill
- **THEN** соответствующая запись переходит в `status='resolved'` без
  повторного вызова `agent_1`

### Requirement: Триграммная статистика для отзывов
Воркер MUST вычислять триграммную статистику текста исключительно для
документов с `document_type='review'`, отдельным Python-скриптом (не
агентом, не тем же процессом, что и аббревиатуры). Результат — список
объектов с текстом триграммы, счётчиком (`count`, положительное целое) и
относительной частотой (`relative_frequency`, число), сохраняемый в
`review_trigrams` (одна строка на `clean_item_id`).

#### Scenario: Документ не является отзывом
- **WHEN** `document_type` документа не равен `review`
- **THEN** триграммная статистика для него не вычисляется, и строка в
  `review_trigrams` не создаётся

#### Scenario: Документ — отзыв
- **WHEN** `document_type` документа равен `review`
- **THEN** для него вычисляется и сохраняется список триграмм с `count` и
  `relative_frequency` в одной строке `review_trigrams`
