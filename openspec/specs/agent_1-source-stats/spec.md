# agent_1-source-stats Specification

## Purpose

Агрегируемая статистика качества источников
(`agent_1_v5.source_stats`, view) — задел под будущий `agent_memory`.

> Синкнуто из change `rework-agent-1-v5` (2026-07-23). Накачено на
> `mvp_db`, sanity-проверка на реальных данных пройдена.

## Requirements

### Requirement: Агрегируемая статистика качества по источникам
По каждому `ID_source` MUST агрегироваться (SQL-view `source_stats`):
`total_raw`, `processed`, `pct_junk`, `pct_non_russian`, `pct_duplicates`,
`avg_content_len`, `last_seen_at`. Статистика вычисляется из `raw_posts` и
вердиктов `clean_posts` (`drop_reason`/`is_duplicate`) без отдельных
лог-таблиц. Процентные доли MUST считаться от `processed` (документов с
вердиктом), а не от `total_raw` — иначе необработанный бэклог искусственно
занижает метрики до завершения бэкфилла.

Это задел под будущую `agent_memory` (приоритизация источников); сама
таблица памяти агентов — вне скоупа.

#### Scenario: Источник с высокой долей мусора
- **WHEN** по источнику накоплены обработанные документы
- **THEN** запрос статистики возвращает его `pct_junk`,
  `pct_non_russian`, `pct_duplicates` и `avg_content_len` одним запросом

#### Scenario: Свежесть источника
- **WHEN** источник давно не отдавал документов
- **THEN** `last_seen_at` отражает время последнего полученного raw-поста

#### Scenario: Необработанный бэклог не искажает метрики
- **WHEN** у источника есть raw-документы без вердикта (`clean_posts`
  ещё не создана)
- **THEN** проценты по этому источнику считаются только по уже
  обработанным документам, а не по всем `total_raw`
