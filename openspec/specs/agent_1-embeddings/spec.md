# agent_1-embeddings Specification

## Purpose

Генерация эмбеддингов для чистых недубликатов `clean_posts`
(`src/agent_1/embed_v5.py`), через OpenRouter (`openai/text-embedding-3-small`,
OpenAI-совместимый `/embeddings`).

> Синкнуто из change `rework-agent-1-v5` (2026-07-23). Провалидировано
> полным прогоном на реальном корпусе: 40314/40314 kept-документов
> заэмбеддены, 0 расхождений по покрытию, идемпотентность подтверждена.

## Requirements

### Requirement: Генерация эмбеддингов для чистых недубликатов
Embedding-воркер MUST генерировать эмбеддинги моделью
`text-embedding-3-small` (через OpenRouter) с логической размерностью
`dimensions=1024` и записывать их в `clean_posts.embedding vector(1024)`.
Эмбеддинг MUST вычисляться только для строк `clean_posts` с
`drop_reason IS NULL` и `is_duplicate=false`.

Так как провайдер вызова (OpenRouter) не документирует параметр
`dimensions` явно, воркер MUST не полагаться на серверную обрезку: он
запрашивает `dimensions=1024`, но в любом случае обрезает ответ до первых
1024 компонент и переприменяет L2-нормировку на своей стороне (эквивалент
серверной Matryoshka-обрезки для `text-embedding-3-*`). Результат MUST
быть вектором единичной длины размерности 1024 независимо от того, что
именно вернул провайдер.

#### Scenario: Чистый недубликат
- **WHEN** строка `clean_posts` прошла очистку и не является дубликатом
- **THEN** воркер вычисляет для неё эмбеддинг и заполняет `embedding`

#### Scenario: Дубликат не эмбеддится
- **WHEN** документ помечен `is_duplicate=true` или отбракован фильтрами
  (`drop_reason IS NOT NULL`)
- **THEN** `embedding IS NULL`, вызов эмбеддера не производится

### Requirement: Батчевые вызовы API
Вызовы эмбеддера MUST выполняться батчами (до лимита API), а не по одному
документу. Claim необработанных строк MUST выполняться через
`FOR UPDATE SKIP LOCKED`, что допускает параллельные воркеры (эмбеддинг —
API-bound задача, в отличие от препроцессинга).

#### Scenario: Пачка чистых документов
- **WHEN** воркер обрабатывает N чистых документов без эмбеддинга
- **THEN** он отправляет их одним или несколькими батч-запросами до лимита
  API, а не N отдельными вызовами

### Requirement: Идемпотентность эмбеддинга
Повторный прогон MUST NOT переэмбеддить уже обработанные документы: при
`embedding IS NOT NULL` вызов эмбеддера пропускается (claim-условие
включает `embedding IS NULL`).

#### Scenario: Повторный прогон на обработанном корпусе
- **WHEN** воркер запускается на корпусе, где у всех чистых недубликатов
  `embedding IS NOT NULL`
- **THEN** выполняется 0 вызовов эмбеддера и 0 изменений строк

### Requirement: HNSW-индекс на clean_posts.embedding
HNSW-индекс (`vector_cosine_ops`) MUST существовать только на
`clean_posts.embedding`; центроиды кластеров/драйверов и эмбеддинги КР
индексами не покрываются (брутфорс быстрее на сотнях строк). При массовой
первичной загрузке индекс MUST создаваться после заливки корпуса, а не до.

#### Scenario: Bulk-загрузка корпуса
- **WHEN** выполняется первичный массовый прогон эмбеддинга корпуса
- **THEN** HNSW-индекс строится после завершения заливки

### Requirement: Эмбеддинг — расходный кэш
Эмбеддинги MUST рассматриваться как производные данные: источник правды —
текст. Допускается обнуление `embedding` у холодных данных при нехватке
места; текст при этом не удаляется, и эмбеддинг восстановим повторным
прогоном.

#### Scenario: Освобождение места
- **WHEN** оператор обнуляет `embedding` у старых строк
- **THEN** `clean_content` и `raw_posts` не затрагиваются, и повторный
  прогон воркера восстанавливает эмбеддинги
